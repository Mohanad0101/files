Сегодня мы изучаем ООП. ООП меняет то, как вы создаете приложения.

вы видете приложение как объекты, которые связаны друг с другом — как в реальной жизни:

    Каждый объект имеет свойства (имя, возраст) и действия (петь, двигаться)

Главные особенности ООП:

    Инкапсуляция – объединяет свойства и действия внутри класса. Можно скрыть их с помощью public, private, protected

    Наследование – дочерний объект получает свойства и действия родительского объекта

    Полиморфизм – одно и то же действие может работать по-разному для разных объектов

Класс – это как форма.
Объект – это как изделие, сделанное по этой форме.

Из одного класса можно создать много объектов.

ООП помогает писать лучший код.
Паттерны проектирования дают готовые решения.

В Java есть разные виды:

    Class : шаблон для создания объектов

    Abstract class: Нельзя создать объект напрямую. Другие классы должны его дополнить

    Static class : принадлежит самому классу. Используется без создания объекта

    Sealed class : определяет, какие классы могут наследовать от него

    Interface : контракт, который говорит, что класс должен делать, но не объясняет как

    Record : простой класс, который в основном хранит данные

    Enum – список фиксированных значений, например дни недели
